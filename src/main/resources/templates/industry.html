Industry Management System

This project is for role base authentication

Screen Time
  • The application is used for industrial purpose and it consist of 5 roles, operator, supervisor and admin
  • Admin has a permission to access operators, supervisors and current users.
  • Supervisor has the permission to access the operator and users.
  • Operator has only the permission for add and view the data.

Role Authorization and Authentication

  • Here we have used ‘role authorization’ and ‘authentication’ methods for accessing and authorize different role to the respective person. Once the role is authorized for the respective role, only authorized person can access the data and permission to access other roles.

Programming and Controllers

  • Admin Controller: To control the supervisor and operator access, update, view and delete.
  • Operator Controller: To control the operator access.
  • Supervisor Controller: To control operator access, update and view the operator purposes.
  • REST API Controller: To use CRUD method for testing users to upload, create and delete the data.
  • MongoDB: For data based initialize, to store the data in database by using class DNInit.
